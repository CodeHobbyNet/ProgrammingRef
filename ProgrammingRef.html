Object Oriented Programming

Basic Concepts

Object Oriented Principles
  • Encapsulation: An object or variable of a class can be hidden from view outside outside the class. Instead of direct access, methods are used to access and modify the objects and variables. This allows the class to place limits on the modification of the objects/variables and protect their integrity.
  • Abstraction: A feature of a class can be represented without explaining the details of what's going on internally. This allows for a simplification of the features and for other classes to use the features more easily.
  • Inheritance: The creation of a new class from another one. The original class is called the base class or super class. The new class that inherits from the base class is called the derived class, sub-class or child class. Inheritance allows for a simplification and reduction of code by allowing code to be inherited from one class to one or more child classes.
  • Polymorphism: The ability to use the same interface for multiple different object types. For example, you can use the same math operators to add integers, floats, doubles, etc.
  • Open recursion: A class can use a special variable, such as this or self to invoke another method or access an object of the current instance of the class.
  • Subtype Polymorphism: A form of Polymorphism in which the a subclass can inherit methods from a superclass. For example, a subclass duck can inherit the fly method from superclass bird and use that method to fly.

C++

  • The Standard Template Library (STL)
Basic C++ Program 
#include <iostream>
class Rectangle
{
	private:
		int width; int height;

	public:
		void setDimensions( int newWidth, int newHeight );
		int getWidth();
		int getHeight();
		int getArea();
};

void Rectangle::setDimensions( int newWidth, int newHeight )
{
	width = newWidth;
	height = newHeight;
}

int Rectangle::getWidth()
{
	return width;
}

int Rectangle::getHeight()
{
	return height;
}

int Rectangle::getArea()
{
	return width*height;
}

int main( int argc, const char* argv[] )
{
	Rectangle r;
	r.setDimensions( 3, 4 );
	std::cout << "Area: " << r.getArea() << std::endl;
	return 0;
}

SQL

SELECT Statement, to retrieve data from the database 
SELECT column_name1, column_name2... FROM table_name WHERE condition;
or
SELECT * FROM table_name WHERE condition;
for selecting all columns

ORDER BY clause, to order the returned data according to a specific column 
SELECT column_name1, column_name2... FROM table_name WHERE condition ORDER BY column_name;

GROUP BY clause, used with aggregate functions to group the aggregate by the values of a column 
SELECT column_name, count(*) FROM table_name WHERE condition GROUP BY column_name;
Will display the count of rows for each distinct value of column_name rather than the count of all rows of the table.

HAVING clause, like the Where clause, but for aggregate functions and GROUP BY 
SELECT column_name, count(*) FROM table_name WHERE condition GROUP BY column_name HAVING count(*) > 10;
Will display the count of rows for each distinct value of column_name that has more than 10 rows.

INSERT, to add rows to a table 
INSERT INTO table_name(column_name1, column_name2, ...) VALUES (value1, value2, ...), (value1, value2, ...);

This is supported by many SQL servers to insert multiple rows with a single INSERT command, but if it doesn't work for your SQL server 
you can always insert one line for every INSERT command like:

INSERT INTO table_name(column_name1, column_name2, ...) VALUES (value1, value2, ...);

UPDATE, to modify a set of rows 
UPDATE table_name SET column_name1 = value1, column_name2 = value2 WHERE condition;

DELETE, to delete a set of rows 
DELETE FROM table_name WHERE condition;

CREATE TABLE, to create a table 
CREATE TABLE table_name(
	column_name1 type1,
	column_name2 type2,
	...
	PRIMARY KEY( column_name1, ...)
);

DROP TABLE, to delete a table 
DROP TABLE table_name;

Data types (defined for a table in the CREATE TABLE statement) (some of these might not be supported by all SQL servers) 
CHARACTER(n) or CHAR(n): fixed length string
CHARACTER VARYING(n) or VARCHAR(n): variable-width string with a maximum size of n characters
BINARY(n): a string of n bits
BINARY VARYING(n) or VARBINARY(n): a variable length string of up to n bits
BOOLEAN: a true or false value
INTEGER, SMALLINT and BIGINT
FLOAT, REAL and DOUBLE PRECISION
NUMERIC(precision, scale) or DECIMAL(precision, scale)
DATE: for date values (e.g. 2011-05-03)
TIME: for time values (e.g. 15:51:36). The granularity of the time value is usually a tick (100 nanoseconds).
TIMESTAMP: This is a DATE and a TIME put together in one variable (e.g. 2011-05-03 15:51:36).

INNER JOIN, to join two or more tables 
SELECT table1.column_name1, table2.column_name2, ...
FROM table1 INNER JOIN table2
ON table1.column1 = table2.column3;

There are different types of join:
INNER JOIN: Returns only rows that the condition is completely met with rows from both tables.
LEFT JOIN (or LEFT OUTER JOIN): Returns all rows from the left table and only rows from the right table that meet the condition.
RIGHT JOIN (or RIGHT OUTER JOIN): Returns all rows from the right table and only rows from the left table that meet the condition.
FULL JOIN (or FULL OUTER JOIN): Returns all rows from either of the tables that match the condition plus all the other rows from each source table.

UNION, to join the results of two or more select statements 
SELECT column_name1, column_name2, ... FROM table1
UNION
SELECT column_name1, column_name2, ... FROM table2

Note 1: UNION returns only distinct values by default, to allow duplicates use UNION ALL instead.

SELECT column_name1, column_name2, ... FROM table1
UNION ALL
SELECT column_name1, column_name2, ... FROM table2

NOTE 2: You can use WHERE clauses and ORDER BY CLAUSES with UNION

SELECT column_name1, column_name2, ... FROM table1 WHERE condition1
UNION ALL
SELECT column_name1, column_name2, ... FROM table2 WHERE condition2
ORDER BY column_name3
