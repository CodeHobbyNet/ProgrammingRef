Object Oriented Programming

Basic Concepts

Object Oriented Principles
  • Encapsulation: An object or variable of a class can be hidden from view outside outside the class. Instead of direct access, methods are used to access and modify the objects and variables. This allows the class to place limits on the modification of the objects/variables and protect their integrity.
  • Abstraction: A feature of a class can be represented without explaining the details of what's going on internally. This allows for a simplification of the features and for other classes to use the features more easily.
  • Inheritance: The creation of a new class from another one. The original class is called the base class or super class. The new class that inherits from the base class is called the derived class, sub-class or child class. Inheritance allows for a simplification and reduction of code by allowing code to be inherited from one class to one or more child classes.
  • Polymorphism: The ability to use the same interface for multiple different object types. For example, you can use the same math operators to add integers, floats, doubles, etc.
  • Open recursion: A class can use a special variable, such as this or self to invoke another method or access an object of the current instance of the class.
  • Subtype Polymorphism: A form of Polymorphism in which the a subclass can inherit methods from a superclass. For exampe, a subclass duck can inherit the fly method from superclass bird and use that method to fly.

C++

#include <iostream>
class Rectangle
{
	private:
		int width; int height;

	public:
		void setDimensions( int newWidth, int newHeight );
		int getWidth();
		int getHeight();
		int getArea();
};

void Rectangle::setDimensions( int newWidth, int newHeight )
{
	width = newWidth;
	height = newHeight;
}

int Rectangle::getWidth()
{
	return width;
}

int Rectangle::getHeight()
{
	return height;
}

int Rectangle::getArea()
{
	return width*height;
}
